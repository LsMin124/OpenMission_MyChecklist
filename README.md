## 나만의 Todo 체크리스트 서비스 개발

### 주제 선정 이유

- 현재 사용중인 Todo List 서비스는 광고가 너무 많이 뜨고, 주기적인 일정 기능이 유료이길래, 해당 서비스를 직접 만들어보려 마음먹음
- 실제로 사용이 가능한 완성된 웹 프로젝트를 해보고 싶었음

### 원하는 목표

- 단순히 Task가 언제까지인지 보고 완료 체크만 하는 체크리스트가 아닌, 
주기적(주간, 월간 등)으로 해야 될 일도 등록하여 처리할 수 있는 체크리스트 구현이 목표
- 주기성 Task 구현을 위해 완료 여부를 각 Task 객체들과 분리함
  - 완료일을 갱신하면 과거 기록이 남지 않음
  - 날짜만 다른 Task를 여럿 생성하면 기간에 따라 DB 낭비가 발생하거나, 수정 시 너무 많은 데이터를 바꿔줘야 함. 

### 기능 구현
#### 1. 회원 및 인증

회원가입 
- [ ] 이메일, 비밀번호, 닉네임 입력
- [ ] 비밀번호  BCrypt 암호화하여 저장
- [ ] 입력값에 대한 Validation (메일 형식, 비밀번호 규칙 등)

로그인
- [ ] ID/PW 검증 후 JWT Access Token 발급

JWT Filter & Security Config
- [ ] 요청 헤더에서 토큰 파싱 및 검증
- [ ] 인증 실패 시 401 Unauthorized 처리할 것

#### 2. Task Management

Task 항목 생성
- [ ] 일회성 Task
- [ ] 주기적 Task
Task 항목 수정/삭제
- [ ] 작성자 본인 화긴
- [ ] 삭제 시 연관된 수행기록 처리 (Cascade 등)

#### 3. 데일리 스케줄 및 수행

오늘의 Task 항목 조회
- [ ] 사용자의 모든 Task 조회
- [ ] 필터링:
  - 일회성: 마감일이 오늘이거나, 마감일이 지났으나 완료가 되지 않은 항목
  - 주기성: 반복 주기 해당 날짜가 오늘인 항목

Task 완료
- [ ] 완료된 Task의 수행 완료 객체 구성
- [ ] 완료된 Task의 예외처리
- [ ] 완료된 항목 취소 시 완료 객체 삭제

#### 4. 통계
- [ ] 월간 수행 현황, 완료율 등
- [ ] 반복 작업의 연속 수행기간 등
